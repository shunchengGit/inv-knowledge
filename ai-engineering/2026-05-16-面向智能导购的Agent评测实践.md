### 核心观点

本文提出一套面向**家居导购 Agent** 的自动化评估链路，基于结构化多维度 Benchmark + LLM-as-a-judge 实现 **91.9% 准确率**的自动评分，并通过人工抽样校准。在"放我家"智能导购应用落地后，量化对比发现 **gpt51 最优**（总分 0.680），较当前线上模型 qwen3-vl **提升 16.4%**。

### 评测链路（四模块闭环）

| 模块 | 作用 |
|------|------|
| **Benchmark 创建** | 构建覆盖典型导购搭配任务的基准测试集 |
| **LLM 模拟人工评测** | 多模态大模型作为"评审员"，维度化判定与打分 |
| **人工抽样验收** | 对 LLM 评审结果抽样复核，计算准确率，校准评分规约 |
| **自动化评测与报告** | 工程化流程，每次模型更新全量评测，自动汇总低分样本与失败模式 |

**执行路径（5步）：** 生成候选搭配结果 → 需求维度拆解 → 多模态自动评测 → 二级维度标注 & 一级维度汇总 → 输出量化结果并问题分析

### 评测维度（两级体系）

| 一级维度 | 二级维度数 | 核心内容 |
|----------|-----------|----------|
| **基础指令维度**（商品选择） | 5项 | 品牌、类目、预算、功能、尺寸 |
| **专业指令维度**（空间设计） | 7项 | 房间类型、结构、分区需求、形态、布局、风格、色彩、材质、工艺 |
| **补充指令维度**（偏好&附加条件） | 6项 | 人群偏好、场景偏好、商品偏好、健康环保、智能需求、使用维护 |
| **用户画像** | - | 典型场景与常见需求的标签化信息 |

### 量化结果

**搭配模型排名（总分）：**

| 模型 | 基础指令 | 专业指令 | 补充指令 | 用户画像 | 总分 |
|------|----------|----------|----------|----------|------|
| **gpt51** | 0.692 | **0.799** | 0.668 | 0.560 | **0.680** |
| gemini25 | 0.681 | 0.766 | 0.668 | **0.581** | 0.674 |
| 外部模型XX | 0.665 | 0.786 | **0.670** | 0.543 | 0.666 |
| qwen3-vl | 0.598 | 0.720 | 0.572 | 0.445 | 0.584 |

- **gpt51** 综合最佳，总分 0.680，优势在基础指令与专业指令
- **qwen3-vl** 四个维度均为最低，与前三名有明显差距

**评测模型准确性（PLCC 相关性）：** gemini25 (0.962) > gpt51 (0.931) > qwen3_vl (0.925) > 外部模型XX (0.896)

### 核心瓶颈（三类共性问题）

1. **无法识别已有家具 → 重复推荐**（频率最高）
2. **未抓住用户核心需求 → 输出跑题**
3. **推荐过量无关商品 → 输出过度**

### 评测准确率

- **整体准确率：91.90%**（1927条评测维度中误判 156 条）
- 误判集中于：尺寸、类目、房间结构、人群偏好、商品偏好、场景偏好

### 六大类误判问题

1. 重复/冲突搭配（未识别已有家具）
2. 类目/功能缺失（关键需求未覆盖）
3. 智能化需求未响应
4. 尺寸不合理
5. 人群/画像不匹配
6. 场景定位或审美偏差

### 可落地建议

- [ ] 建立多维度 Benchmark 覆盖真实业务场景
- [ ] 采用 LLM-as-a-judge + 人工抽样校准的混合评测模式
- [ ] 使用 gpt51 作为搭配基座模型（当前最优）
- [ ] 使用 gemini25 作为评测模型（稳定性最高）
- [ ] 优先解决"识别已有家具"问题（最高频瓶颈）
- [ ] 优化需求抽取能力，避免输出跑题
- [ ] 增加约束控制，减少过度推荐
